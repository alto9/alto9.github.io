Overview

kube9 Operator is a Kubernetes operator that extends and enhances the capabilities of the kube9 VS Code Extension. It runs inside your Kubernetes cluster and provides advanced automation, monitoring, and management features that complement the client-side extension.

By deploying the kube9 Operator in your cluster, you unlock:

  • Automated resource optimization and recommendations
  • Continuous security scanning and compliance monitoring
  • Custom resource definitions (CRDs) for kube9-specific workflows
  • Enhanced AI analysis with direct cluster access
  • Team collaboration features and shared insights

Planned Features

Custom Resource Definitions

The operator introduces several CRDs to manage kube9-specific workflows:

  • kube9Analysis: Stores AI analysis results for cluster resources
  • kube9Optimization: Defines optimization recommendations and tracks application
  • kube9SecurityScan: Manages security scanning schedules and results
  • kube9Policy: Defines organizational policies and compliance rules
  • kube9Insight: Stores team-shared insights and recommendations

Getting Started

Prerequisites: kube9 Operator requires a running Kubernetes cluster (1.20+) and kubectl configured for cluster access.

Installation

Once released, you'll be able to install kube9 Operator using Helm:

# Add the kube9 Helm repository
helm repo add kube9 https://charts.kube9.dev
helm repo update

# Install the operator
helm install kube9-operator kube9/kube9-operator \
  --namespace kube9-system \
  --create-namespace

# Verify installation
kubectl get pods -n kube9-system

Configuration

Configure the operator through the Helm values file:

# values.yaml
replicaCount: 1

operator:
  # Enable AI analysis features
  aiAnalysis:
    enabled: true
    interval: 5m
  
  # Security scanning configuration
  securityScanning:
    enabled: true
    schedule: "0 */6 * * *"
  
  # Optimization features
  optimization:
    enabled: true
    autoApply: false  # Require manual approval

# Connection to kube9 Server
kube9Server:
  url: "https://api.kube9.dev"
  # API key stored as secret

# Resource limits
resources:
  limits:
    cpu: 200m
    memory: 256Mi
  requests:
    cpu: 100m
    memory: 128Mi

Integration with VS Code Extension

The VS Code extension automatically detects when the operator is installed in your cluster and enables additional features:

  1. Open kube9 in VS Code
  2. Connect to a cluster with the operator installed
  3. Look for the "Operator Enabled" badge in the cluster view
  4. Access operator features through the context menu
