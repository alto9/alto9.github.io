Our Commitment: We believe in complete transparency about data collection. This page provides full details about what data we collect, how we use it, and how we protect it.

What Data We Collect

The kube9-operator collects sanitized, anonymized cluster metrics to power AI-powered features. We collect only what's necessary and never collect sensitive information.

Data Categories

Category What's Collected Frequency Purpose
Cluster Metadata Kubernetes version, node count, provider, region Every 24 hours Service provisioning, compatibility checks
Resource Inventory Counts of pods, deployments, services (no names or details) Every 6 hours Understanding cluster scale and complexity
Configuration Patterns Sanitized resource specs (names/IPs removed, values hashed) Every 12 hours AI model training, pattern recognition
Performance Metrics Aggregated CPU/memory utilization (no individual pod data) Every 15 minutes Performance insights and recommendations
Security Posture RBAC policy counts, network policy coverage percentages Every 24 hours Security recommendations and compliance

How Data Is Sanitized

All data is sanitized before it leaves your cluster. Our sanitization process includes:

Sanitization Steps

  1. Remove PII: All names, IPs, and identifying information removed
  2. Hash Sensitive Values: Environment variable values hashed (keys preserved)
  3. Normalize Data: Round numeric values, convert timestamps to relative time
  4. Extract Patterns: Extract configuration patterns without raw data
  5. Validate: Automated checks ensure no sensitive data escapes
Privacy by Design: Sanitization happens in your cluster, before any data is transmitted. This ensures your sensitive information never leaves your environment.
