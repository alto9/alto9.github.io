1. Introduction

Alto9 ("we", "our", or "us") operates the kube9-operator, a Kubernetes operator that enables AI-powered features in the Kube9 VS Code extension. This Privacy Policy explains how we collect, use, disclose, and safeguard your information when you use the kube9-operator.

By using the kube9-operator, you agree to the collection and use of information in accordance with this policy. If you do not agree with our policies and practices, do not use the kube9-operator.

2. Information We Collect

2.1 Cluster Data

When you install the kube9-operator in your Kubernetes cluster, we collect sanitized, anonymized cluster metrics. This includes:

  • Cluster metadata (Kubernetes version, node count, provider, region)
  • Resource inventory (counts of pods, deployments, services)
  • Configuration patterns (sanitized resource specifications)
  • Performance metrics (aggregated CPU/memory utilization)
  • Security posture indicators (RBAC policy counts, network policy coverage)

For detailed information about what we collect, see our Data Collection page.

2.2 What We Never Collect

We explicitly do not collect:

  • Resource names in original form (replaced with {resource-type}-{hash} format, e.g., pod-a1b2c3d4e5f6)
  • IP addresses or network endpoints
  • Secret contents or sensitive values
  • Environment variable values (only keys, values are hashed)
  • User credentials or tokens
  • Application logs or container output
  • Any personally identifiable information (PII)

2.3 Account Information

If you use Pro tier features, we collect:

  • Account information (email address, name) - via Kube9 Portal
  • API keys (stored securely, never logged)
  • Billing information (processed by third-party payment processors)

3. How We Use Your Information

3.1 Primary Uses

  • Service Delivery: Provide AI-powered features in the VS Code extension
  • AI Model Training: Train proprietary AI models on Kubernetes patterns (fully anonymized)
  • Service Improvement: Understand usage patterns to improve features
  • Business Intelligence: Aggregated analytics for product development

3.2 We Never

  • Sell or share your data with third parties
  • Use your data to compete with you
  • Identify individual applications or users
  • Access your cluster directly (all communication is outbound)

4. Data Sanitization

All data is sanitized before it leaves your cluster. Our sanitization process includes:

  • Hashing all resource names with {resource-type}-{hash} format (e.g., pod-a1b2c3d4e5f6). Same name always produces same hash, allowing relationship tracking without revealing names.
  • Removing all IPs and identifying information
  • Hashing sensitive values (environment variables, etc.)
  • Normalizing data (rounding numbers, relative timestamps)
  • Extracting patterns without raw data
  • Automated validation to ensure no sensitive data escapes

For detailed information about sanitization, see our Data Collection page.

5. Data Sharing and Disclosure

5.1 We Do Not Sell Data

We do not sell, trade, or rent your data to third parties.

5.2 Service Providers

We may share data with service providers who assist us in operating our service:

  • Cloud Providers: AWS, GCP, Azure for hosting (data remains encrypted)
  • Payment Processors: For billing (they do not receive cluster data)
  • Analytics: Aggregated, anonymized analytics only

All service providers are contractually obligated to protect your data.

5.3 Legal Requirements

We may disclose data if required by law or in response to valid legal requests.

6. Data Retention

We retain data only as long as necessary:

Data Type Retention Period
Raw Collected Data 90 days
Sanitized Metrics 2 years
Aggregated Analytics 5 years
Audit Logs 7 years

You can request deletion of your data at any time. See Section 8 for details.

7. Data Security

We implement industry-standard security measures to protect your data:

  • Encryption in Transit: All data encrypted using HTTPS/TLS
  • Encryption at Rest: All data encrypted using AES-256
  • No Ingress: Operator only makes outbound connections
  • API Key Security: API keys stored securely, never logged
  • Access Controls: Restricted access to data on a need-to-know basis
  • Regular Audits: Security audits and penetration testing

8. Your Rights

You have the following rights regarding your data:

8.1 Right to Access

You can request access to what data we collect about your cluster. Contact us at privacy@kube9.io.

8.2 Right to Correction

You can request correction of inaccurate data.

8.3 Right to Deletion

You can request deletion of your data ("right to be forgotten"). Data will be deleted within 30 days (with grace period for recovery).

8.4 Right to Opt-Out

You can opt-out of data collection by:

  • Installing the operator without an API key (Free tier, no data collection)
  • Contacting us to disable data collection (some Pro features may be limited)

8.5 Right to Portability

You can request export of your data in machine-readable format (JSON, CSV).

8.6 Right to Complain

You can file complaints with regulatory authorities if you believe we have violated your privacy rights.

To Exercise Your Rights: Contact us at privacy@kube9.io or through the Kube9 Portal.
