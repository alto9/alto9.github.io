Overview

Kandy Operator is a Kubernetes operator that extends and enhances the capabilities of the Kandy VS Code Extension. It runs inside your Kubernetes cluster and provides advanced automation, monitoring, and management features that complement the client-side extension.

By deploying the Kandy Operator in your cluster, you unlock:

  • Automated resource optimization and recommendations
  • Continuous security scanning and compliance monitoring
  • Custom resource definitions (CRDs) for Kandy-specific workflows
  • Enhanced AI analysis with direct cluster access
  • Team collaboration features and shared insights

Custom Resource Definitions

The operator introduces several CRDs to manage Kandy-specific workflows:

  • KandyAnalysis: Stores AI analysis results for cluster resources
  • KandyOptimization: Defines optimization recommendations and tracks application
  • KandySecurityScan: Manages security scanning schedules and results
  • KandyPolicy: Defines organizational policies and compliance rules
  • KandyInsight: Stores team-shared insights and recommendations

Getting Started

â„¹ï¸ Prerequisites: Kandy Operator requires a running Kubernetes cluster (1.20+) and kubectl configured for cluster access.

Installation

Once released, you'll be able to install Kandy Operator using Helm:

# Add the Kandy Helm repository
helm repo add kandy https://charts.kandy.dev
helm repo update

# Install the operator
helm install kandy-operator kandy/kandy-operator \
  --namespace kandy-system \
  --create-namespace

# Verify installation
kubectl get pods -n kandy-system

Configuration

Configure the operator through the Helm values file:

# values.yaml
replicaCount: 1

operator:
  # Enable AI analysis features
  aiAnalysis:
    enabled: true
    interval: 5m
  
  # Security scanning configuration
  securityScanning:
    enabled: true
    schedule: "0 */6 * * *"
  
  # Optimization features
  optimization:
    enabled: true
    autoApply: false  # Require manual approval

# Connection to Kandy Server
kandyServer:
  url: "https://api.kandy.dev"
  # API key stored as secret

# Resource limits
resources:
  limits:
    cpu: 200m
    memory: 256Mi
  requests:
    cpu: 100m
    memory: 128Mi

Integration with VS Code Extension

The VS Code extension automatically detects when the operator is installed in your cluster and enables additional features:

  1. Open Kandy in VS Code
  2. Connect to a cluster with the operator installed
  3. Look for the "Operator Enabled" badge in the cluster view
  4. Access operator features through the context menu
