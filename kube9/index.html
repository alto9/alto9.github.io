Overview

Kube9 is an open source toolkit for Kubernetes Well-Architected Framework validation and intelligent cluster management. The toolkit consists of a Kubernetes operator (the core component), a VS Code extension, and optional Helm-based UI components.

The kube9-operator runs in your cluster, performing scheduled Well-Architected Framework assessments across all 6 pillars. In Free tier mode, it generates point-in-time reports. With an API key configured, it establishes scheduled data reporting to kube9-server, enabling Pro tier insights and continuous compliance tracking.

The VS Code extension and optional UI components provide access to framework assessments and cluster management capabilities. The extension can also work without the operator for basic kubectl operations.

Features

Multi-Cluster Management

Import and manage multiple kubeconfig files. Switch between clusters with ease and view all your Kubernetes environments in one place.

Proactive AI Insights

AI continuously monitors your clusters and automatically surfaces issues before they become critical. 24 automated insights per day on Pro tier find problems you didn't know existed.

Real-Time Visualization

View pods, services, deployments, and other resources in real-time with an intuitive tree view interface integrated directly into VS Code.

Context-Aware Analysis

Click any resource to get instant AI analysis. Understand why deployments fail, why pods consume high resources, and get actionable recommendations - all without leaving your IDE.

Intelligent Troubleshooting

Get AI-generated explanations for failures and errors with actionable recommendations to resolve issues quickly.

Zero-Trust Security

Name obfuscation ensures your resource names never leave your cluster. All communication is egress-only - no cluster ingress required. Your kubeconfig stays on your machine.

ArgoCD Integration

Seamless integration with ArgoCD for enhanced drift detection and GitOps visibility. Pro tier insights based on ArgoCD deployment data when available in your cluster.

Getting Started

Installation

For Users

  1. Open VS Code
  2. Go to Extensions (Ctrl/Cmd + Shift + X)
  3. Search for "kube9"
  4. Click Install

For Developers

# Clone the repository
git clone https://github.com/alto9/kube9.git
cd kube9
# Install dependencies (requires Node.js 22+)
npm install
# Compile TypeScript
npm run compile
# Run in development mode
# Press F5 in VS Code to open Extension Development Host

Configuration

Configure Kube9 in your VS Code settings:

{
"kube9.apiKey": "kdy_prod_abc123def456",
"kube9.serverUrl": "https://api.kube9.io",
"kube9.debugMode": false
}

First Steps

  1. Open the kube9 view in the Activity Bar (sidebar)
  2. Import your kubeconfig file: Ctrl/Cmd + Shift + P ‚Üí "kube9: Import kubeconfig"
  3. Select a cluster from the kube9 sidebar
  4. Start exploring your cluster or ask AI for analysis

Architecture

Proactive AI Architecture

Kube9 uses a zero-trust architecture with proactive background analysis:

VS Code Extension

  • UI & Tree Views
  • Local Cluster Access
  • Insight Display
  • User Interactions

Kube9 Operator

  • Metrics Collection
  • Name Obfuscation
  • Hourly Polling
  • Insight De-obfuscation

Kube9 Server

  • Background AI Analysis
  • LangChain Orchestration
  • Insight Generation
  • Caching & Storage

How It Works

  1. Operator collects metrics - Sanitizes data and obfuscates resource names
  2. Metrics sent to server - Egress-only, no waiting for AI processing
  3. Background AI analysis - LangChain orchestrates multi-step reasoning on server
  4. Insights generated & cached - Stored with obfuscated names for security
  5. Operator polls hourly - Retrieves cached insights, no real-time AI calls
  6. Local de-obfuscation - Operator maps mock names back to real resource names
  7. Extension displays insights - Reads from local operator, shows in VS Code

Zero-Trust Security Model

  • Name Obfuscation: Resource names replaced with mock names before leaving cluster
  • Egress-Only: No cluster ingress required - operator initiates all communication
  • Local De-obfuscation: Real names never leave cluster, mapping stays in operator
  • Secure Storage: API keys stored in VS Code's secret storage
  • Audit Logging: All AI interactions logged for compliance and debugging
