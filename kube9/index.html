Overview

kube9 transforms Kubernetes management from complex command-line operations into an intuitive, AI-assisted experience. Built as a VS Code extension, kube9 brings the power of AI directly into your development environment for seamless cluster operations.

Instead of memorizing kubectl commands and YAML syntax, simply ask questions in natural language:

"Show me pods using high CPU"
"Why is my deployment failing?"
"Suggest resource limits for my Node.js app"
"Check security vulnerabilities"

Features

Multi-Cluster Management

Import and manage multiple kubeconfig files. Switch between clusters with ease and view all your Kubernetes environments in one place.

Natural Language Queries

Ask questions about your cluster in plain English. kube9's AI understands your intent and provides intelligent insights and recommendations.

Real-Time Visualization

View pods, services, deployments, and other resources in real-time with an intuitive tree view interface integrated directly into VS Code.

Context-Aware Assistance

AI understands your cluster state and provides relevant suggestions based on current conditions and best practices.

Intelligent Troubleshooting

Get AI-generated explanations for failures and errors with actionable recommendations to resolve issues quickly.

Secure by Design

Hybrid architecture keeps your cluster data secure. Sensitive information stays in your environment while leveraging AI capabilities.

Getting Started

Installation

For Users

  1. Open VS Code
  2. Go to Extensions (Ctrl/Cmd + Shift + X)
  3. Search for "kube9"
  4. Click Install

For Developers

# Clone the repository
git clone https://github.com/alto9/kube9.git
cd kube9
# Install dependencies (requires Node.js 22+)
npm install
# Compile TypeScript
npm run compile
# Run in development mode
# Press F5 in VS Code to open Extension Development Host

Configuration

Configure kube9 in your VS Code settings:

{
"kube9.apiKey": "kdy_prod_abc123def456",
"kube9.serverUrl": "https://api.kube9.dev",
"kube9.debugMode": false
}

First Steps

  1. Open the kube9 view in the Activity Bar (sidebar)
  2. Import your kubeconfig file: Ctrl/Cmd + Shift + P → "kube9: Import kubeconfig"
  3. Select a cluster from the kube9 sidebar
  4. Start exploring your cluster or ask AI for analysis

Architecture

Hybrid AI Architecture

kube9 uses a hybrid architecture that balances security, performance, and user experience:

VS Code Extension

  • UI Management & Tree Views
  • Cluster Communication
  • Data Collection
  • Recommendation Display

kube9 Server

  • AI Service Integration
  • Prompt Engineering
  • Response Processing
  • Caching & Optimization

Data Flow

  1. Extension collects cluster data and user context
  2. Extension sends structured request to kube9 Server
  3. Server processes data and formats AI prompts
  4. Server calls AI service (OpenAI, Anthropic, etc.)
  5. Server processes and validates AI response
  6. Server returns formatted recommendations to Extension
  7. Extension displays recommendations in webview panels

Security Model

  • Data Sanitization: Remove sensitive information before AI processing
  • Request Validation: Validate all requests from extension
  • Secure Storage: API keys stored in VS Code's secret storage
  • Audit Logging: Log all AI interactions for debugging and compliance
