Overview

The Kubernetes Well-Architected Framework provides a comprehensive set of best practices for designing and operating reliable, secure, high-performing, and cost-effective Kubernetes clusters. Similar to the AWS Well-Architected Framework, this framework helps you understand the pros and cons of decisions you make while building and operating Kubernetes workloads.

Kube9 implements and audits this framework through a dual-tier architecture that combines Free tier validation with Pro tier analysis, providing comprehensive coverage while maintaining data privacy.

Dual-Tier Architecture

Kube9 uses both Free tier checks (deterministic validation) and Pro tier analysis (pattern detection and AI insights) to provide comprehensive framework coverage.

Free Tier: Deterministic checks that can be validated directly (e.g., "does this service have a network policy?", "does this certificate expire in 14 days?"). Available without an API key.

Pro Tier: Pattern detection, trend analysis, anomaly detection, and complex reasoning across multiple resources or time periods. Requires API key for scheduled data reporting.

Dual-Tier Architecture

Kube9 implements the Well-Architected Framework through a dual-tier architecture that combines Free tier validation with Pro tier analysis. This approach provides comprehensive coverage while maintaining data privacy.

Free Tier

  • Location: Runs inside your Kubernetes cluster
  • Access: Full access to real resource names and configurations
  • Privacy: All checks happen locally, results stored in CRDs
  • Capabilities: CVE scanning, compliance checks, configuration validation
  • Requires: Real names (e.g., "nginx:1.14.0" for CVE lookups)
  • API Key: Not required

Pro Tier

  • Location: Runs on kube9-server
  • Access: Obfuscated data only (mock names)
  • Privacy: Cannot see real names, cannot perform CVE lookups
  • Capabilities: Pattern detection, trend analysis, Pro tier recommendations
  • Works With: Configuration patterns (preserved despite obfuscation)
  • API Key: Required for scheduled data reporting

Why the Split?

Free Tier: Deterministic validation that requires real resource names or direct cluster access (e.g., CVE scanning needs "nginx:1.14.0", certificate expiration needs actual dates). These are simple rule-based checks that don't require AI.

Pro Tier: Pattern detection, trend analysis, and complex reasoning that works with obfuscated data because configuration patterns (service types, ports, security contexts) are preserved. AI identifies anomalies, predicts trends, and provides context-aware recommendations that go beyond simple rule checking.
