Overview

Synth9 bridges the gap between your local CDK/SAM code and deployed CloudFormation stacks. No more navigating AWS Console or manually matching stack names‚Äîright-click your stack code to instantly view resources, access Lambda logs, and debug without leaving VS Code.

The Problem: When debugging issues in dev or staging environments, finding the right CloudFormation stack and accessing its resources (especially Lambda logs) requires:

  • Remembering or guessing deployed stack names
  • Navigating through AWS Console or AWS Explorer
  • Manually matching code constructs to deployed resources
  • Switching between multiple environments

The Solution: Synth9 automatically:

  • Synthesizes your CDK/SAM code to discover stack names
  • Matches code to deployed CloudFormation stacks
  • Shows all resources in a visual tree
  • Provides one-click access to logs and metrics

Getting Started

Prerequisites

  • VS Code 1.80.0+
  • AWS CLI installed and configured
  • CDK CLI installed (for CDK projects) or SAM CLI (for SAM projects)
  • AWS credentials configured (~/.aws/credentials or environment variables)

Installation

For Users

  1. Open VS Code
  2. Go to Extensions (Ctrl/Cmd + Shift + X)
  3. Search for "Synth9"
  4. Click Install

For Developers

# Clone the repository
git clone https://github.com/alto9/synth9.git
cd synth9
# Install dependencies (requires Node.js 22+)
npm install
# Compile TypeScript
npm run compile
# Run in development mode
# Press F5 in VS Code to open Extension Development Host

Quick Start

‚Üí Open a CDK or SAM project in VS Code
‚Üí Right-click on cdk.json, a stack definition file, or a Lambda construct
‚Üí Select "Synth9: Show Stack Resources"
‚Üí If multiple matching stacks exist, choose the environment you want
‚Üí Explore resources in the tree view

Configuration

Configure Synth9 in your VS Code settings:

{
// Cache duration for synthesized stack names (in minutes)
"synth9.cacheDuration": 60,
// AWS profile to use (leave empty to use default)
"synth9.awsProfile": "",
// AWS region to use (leave empty to use profile default)
"synth9.awsRegion": ""
}

How It Works

Architecture

Synth9 runs entirely locally using AWS CLI and CDK/SAM CLI:

VS Code Extension

  • Discovers CDK/SAM projects
  • Synthesizes stack names
  • Queries CloudFormation via AWS CLI
  • Displays resource tree
  • Provides quick actions

AWS CLI Integration

  • List CloudFormation stacks
  • Describe stack resources
  • Access CloudWatch Logs
  • Query CloudWatch Metrics

Stack Matching Process

  1. Discovery: Find all cdk.json files in workspace
  2. Synthesis: Run cdk synth to get deterministic stack names
  3. Query: Use AWS CLI to list CloudFormation stacks
  4. Match: Match synthesized names to deployed stacks (handles CDK's random suffixes)
  5. Resolve: If multiple matches, show picker for user selection
  6. Display: Show all resources in selected stack

Example Workflow

Your CDK code defines:
new Stack(this, 'MyStack', { ... })
CDK synthesizes to:
"MyApp-MyStack"
Deployed stacks might be:
"MyApp-MyStack-Dev-ABC123"
"MyApp-MyStack-Prod-XYZ789"
Synth9 matches and shows:
"Found 2 matching stacks:
[1] MyApp-MyStack-Dev-ABC123 (Last updated: 2 hours ago)
[2] MyApp-MyStack-Prod-XYZ789 (Last updated: 1 day ago)
  
Which stack do you want to view?"

How It Differs from AWS Toolkit

While AWS Toolkit provides general AWS resource management, Synth9 focuses specifically on:

  • Code-to-Stack Mapping: Automatically connects your code to deployed stacks
  • Multi-Environment Workflow: Easily switch between dev/staging/prod stacks
  • Resource Discovery: Shows all resources created by a stack, not just individual services
  • Quick Debugging: Fast access to logs from your code without manual navigation
